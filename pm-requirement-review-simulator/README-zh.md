# PM 需求评审模拟器

[English](README.md)

> 模拟真实跨部门评审会议的攻防推演技能。五方角色（技术/运营/设计/老板/法务）轮番质疑，输出带存活率评分的 HTML 可视化报告。

## 适用场景

- **评审前预演**：提前暴露需求盲区，带着应对方案进会议室
- **PRD 自检**：用确定性评分引擎给需求文档做"体检"
- **新人练兵**：模拟大厂评审会压力，练习应对尖锐质疑
- **展示版 PRD 评测**：对公开发表的 PRD 文章做专业度评估

## 你会得到什么

一份**浅蓝风格 HTML 报告**（浅蓝渐变 Hero + 白色卡片 + SVG 仪表盘与雷达图），包含：

- **存活率评分**——确定性评分引擎（逐项 0/1/2 加权公式），不靠感觉
- **五维雷达图**——可视化展示强项和致命弱点
- **决策闸门**——价值/风险/资源/战略闸门 + A/B/C 方案对比
- **杀手回复 TOP 3**——最难的 3 个问题，"普通回复" vs "杀手回复" + 技巧拆解
- **RACI 矩阵**——跨团队协作方案，含冲突识别与解决
- **会议脚本**——可直接使用的开场 → 核心论证 → 风险应对 → 决策 → 收尾
- **行动清单**——责任人 + 截止时间 + 交付物 + 回看节点

## 三档残酷度

| 档位 | 风格 | 适合谁 |
|------|------|--------|
| 🟢 新手村 | 温和建议，建设性语气 | 新人产品经理，第一次练习 |
| 🟡 实战 | 大厂评审会标准强度 | 评审前预演 |
| 🔴 地狱 | 全员敌对 + 行业黑话攻击 | 资深产品经理压测边界场景 |

## 快速启动

```text
帮我评审一下我们要做的拼团功能，实战模式。
```

技能会先输出「信息采集清单」引导补充，确认后生成完整 HTML 报告。详细输入模板见 `references/user_templates.md`。

## 工作流

```
提交需求 → 信息采集清单 → 五角色质疑 + 打分 → HTML 报告
```

1. **输入**：识别需求类型（重大功能 / 工具 / MVP / 迭代 / 合规）+ PRD 来源
2. **处理**：五角色按人设质疑 → 逐项打分 → 权重归一化 → 存活率
3. **输出**：生成 11 章节 HTML 报告（浅蓝单一主题）

## 报告章节

| # | 章节 | 说明 |
|---|------|------|
| 1 | Hero 封面 | 需求概览、评级徽章、决策标签 |
| 2 | Radial Hub | 存活率仪表盘 + 五维雷达图 |
| 3 | Callouts | 致命死穴 / 救命稻草高亮 |
| 4 | 决策闸门 + A/B/C 方案 | 价值/风险/资源/战略闸门 + 方案对比 |
| 5 | 五方质疑面板 | 按角色分组，按致命/重要/次要排序 |
| 6 | 杀手回复 TOP 3 | 普通回复 vs 杀手回复 + 技巧分析 |
| 7 | 跨团队协作 | RACI 矩阵 + 冲突识别与解决 |
| 8 | 评审会议脚本 | 开场 → 核心论证 → 风险预案 → 决策 → 收尾 |
| 9 | 优化建议 | 🔴 必修 / 🟡 可选 / ⚪ 延后 |
| 10 | 行动清单 | 责任人 + 截止时间 + 交付物 + 回看节点 |
| 11 | 免责声明 | AI 生成提示 + 适用边界 |
